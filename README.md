# 📖 Overview

This repository is a structured collection of my accepted **AtCoder** solutions.

Rather than storing only accepted source code, every problem is documented to capture the complete reasoning behind the solution—from identifying the underlying algorithm to analyzing its complexity and recording important implementation details.

The repository is designed to serve three purposes:

- Personal revision and knowledge retention
- Demonstrating problem-solving ability through clean implementations
- Maintaining a high-quality competitive programming archive

---

# 🏗 Repository Design

The repository follows the same structure for every contest and every problem.

```text
AtCoder-Solutions/
│
├── AtCoder-Beginner-Contest/
│   └── ABC001/
│       └── A - Welcome to AtCoder/
│           ├── solution.cpp
│           └── README.md
│
├── AtCoder-Regular-Contest/
├── AtCoder-Grand-Contest/
├── Educational-DP-Contest/
│
└── Library/
```

This consistent organization makes navigation, revision, and maintenance straightforward as the repository grows.

---

# 📝 Documentation Standard

Every solved problem follows the same documentation template.

```text
Problem/
├── solution.cpp
└── README.md
```

Each problem README includes:

- 🔗 Official Problem Link
- 📄 Problem Overview
- 💡 Solution Approach
- ⚙️ Algorithm
- 📊 Time & Space Complexity
- ⚠️ Edge Cases
- 📚 STL Used
- 🛠 Implementation Notes
- 🎯 Key Takeaways

The goal is to explain **why** the solution works, not just present accepted code.

---

# 💻 Coding Standards

Every implementation in this repository follows a consistent engineering style.

- Modern C++17
- Fast I/O where appropriate
- Readable and maintainable code
- Meaningful variable and function names
- Minimal use of macros
- Standard Library preferred whenever suitable
- Complexity analysis included
- Comments only where they improve understanding
